# Big-O Notation

## TL;DR
> Big-O describes how an algorithm's running time grows as the input grows, ignoring constants.

## Common classes (fast to slow)
- O(1) — constant
- O(log n) — logarithmic
- O(n) — linear
- O(n log n) — linearithmic
- O(n^2) — quadratic
